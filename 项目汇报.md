# 基于 YOLOv5 的智能门禁系统 — 项目汇报

## 一、项目概述

本项目是一个**基于 YOLOv5 人脸检测的智能门禁系统**，使用 Python + PyQt5 + OpenCV 开发，搭载于**龙芯 2K1000 (MIPS64el) 嵌入式平台**。系统集成 YOLOv5 深度学习人脸检测、LBPH 人脸识别、超声波测距、红外体温测量、语音播报、道闸控制等功能，实现非接触式身份验证与健康筛查一体化。

**核心流程：** 人脸录入 → 模型训练 → YOLOv5 检测 → LBPH 识别 → 体温检测 → 语音报警 → 道闸控制

**总代码量：** Python ~3355行 + C驱动 ~1359行 = **约 4700 行**

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 深度学习 | YOLOv5 (Ultralytics) | 人体检测，FPS≥10 |
| 模型部署 | ONNX Runtime | 龙芯平台推理方案 |
| 人脸识别 | OpenCV LBPH | 纹理特征提取 + 置信度比对 |
| UI 框架 | PyQt5 | 三标签页 + 多线程信号/槽 |
| 图像采集 | OpenCV 4.13 | USB 摄像头驱动 |
| 语音合成 | Windows: SAPI / Linux: pyttsx3+espeak | 平台自适应 |
| 数据库 | SQLite3 | 用户管理 + 人脸特征 + 通行记录 |
| 硬件驱动 | C语言 .so 动态库 | GPIO/PWM/超声波/红外 |
| 目标平台 | 龙芯 2K1000 (MIPS64el) / Linux | 国产化嵌入式部署 |
| 开发平台 | Windows 11 (Anaconda) | 模拟模式调试 |

### 4.1 YOLOv5 人脸检测（四级回退策略）

[core/face_detector.py](core/face_detector.py) 实现了从高到低四级检测方案：

```
┌─────────────────────────────────────────────┐
│ 检测优先级 (FaceDetector._init_backends)     │
│                                             │
│ 1. ONNX Runtime  → 龙芯2K1000 部署方案      │
│ 2. Ultralytics   → Windows 开发环境 (当前)  │
│ 3. PyTorch Hub   → 传统 torch.hub 方式      │
│ 4. Haar 级联      → 通用后备，无需 GPU       │
└─────────────────────────────────────────────┘
```

**检测流程**（Haar 优先，YOLOv5 回退，确保与录入方式一致）：

```python
# main.py:432 — AutoRunThread 人脸检测
def _detect_faces_haar_first(self, img, ...):
    # 1. Haar 级联精确检测人脸特征（眼部/鼻/嘴）
    haar_faces = self.face_cascade.detectMultiScale(
        gray, scaleFactor=1.3, minNeighbors=5, minSize=(60, 60))
    # 2. 过滤 ROI 区域
    for (fx, fy, fw, fh) in haar_faces:
        if fx > x_min and fy > y_min and fx + fw < x_max ...:
            faces.append((fx, fy, fw, fh, 1.0))
    # 3. Haar 没检测到 → YOLOv5 回退
    if not faces:
        faces, _ = self.detector.detect_in_roi(img, ...)
    return faces
```

**YOLOv5 推理代码**（Ultralytics 后端）：

```python
# core/face_detector.py:209 — Ultralytics 推理
def _detect_ultralytics(self, img):
    results = self.ultralytics_model(img, conf=self.confidence, verbose=False)
    for i, (box, conf, cls_id) in enumerate(zip(boxes, confs, cls_ids)):
        if int(cls_id) == 0:  # COCO class 0 = person
            # 从人体框估算面部区域（上方 35%）
            face_x, face_y = int(x1 + fw * 0.2), int(y1)
            face_w, face_h = int(fw * 0.6), int(fh * 0.35)
            faces.append((face_x, face_y, face_w, face_h, float(conf)))
```

### 4.2 LBPH 人脸识别

[core/face_recognizer.py](core/face_recognizer.py) 实现基于 LBPH 纹理直方图的人脸特征提取与比对。

**识别流程：**

```
录入阶段          训练阶段             识别阶段
┌─────────┐     ┌──────────┐       ┌──────────┐
│ Haar检测 │ →  │ LBPH训练  │  →   │ Haar检测  │
│ 20张人脸 │     │ model.xml │       │ 裁剪 ROI  │
│ 缩放200² │     │ 特征存DB  │       │ 缩放200²  │
└─────────┘     └──────────┘       │ LBPH预测  │
                                    │ 置信度<70→通过│
                                    └──────────┘
```

```python
# core/face_recognizer.py:195 — LBPH 预测
def predict(self, face_roi):
    gray = cv2.resize(face_roi, (200, 200))   # 缩放匹配训练数据
    label, confidence = self.lbph.predict(gray) # LBPH 直方图匹配
    if confidence < 70 and label < len(self.names):
        return self.names[label], confidence    # 识别成功
    return None, confidence                     # 陌生人
```

**关键设计决策：** 录入和识别都使用**同一套 Haar 级联**检测人脸，确保人脸 ROI 裁剪区域一致，LBPH 模型才能正确匹配。

## 六、关键技术问题与解决方案

### 6.1 YOLOv5 录入与识别检测不一致

**问题：** 录入时 Haar 级联精确检测人脸特征，但自动运行时 YOLOv5 从"person"全身框估算上方 35% 作为人脸，裁剪区域不匹配，LBPH 无法正确识别。

**解决：** [main.py:432](main.py) 全部识别线程统一为 **Haar 优先 + YOLOv5 回退**策略，确保录入和识别使用同一检测方式。

### 6.2 pyttsx3 Windows SAPI 引擎复用 bug

**问题：** pyttsx3 的 Windows SAPI 驱动在第一次 `runAndWait()` 后引擎内部停止，后续 `say()` 静默失败。多线程共享引擎导致状态错乱。

**解决：** [main.py:51](main.py) 平台自适应 — Windows 直连 `SAPI.SpVoice`，Linux/龙芯用 pyttsx3 + espeak（无此 bug）。

### 6.3 SQLite 外键约束导致训练失败

**问题：** `face_features.user_id` 有外键 `REFERENCES users(id)`，录入的人名不在 users 表时 `user_id=0` 违反约束。

**解决：** [core/database.py:36](core/database.py) 移除外键约束，`user_id` 改为可空。

### 6.4 中文路径支持

**问题：** OpenCV `cv2.imread`/`cv2.imwrite` 不支持 Unicode 路径。

**解决：** [utils/face_utils.py:17](utils/face_utils.py) 使用 `cv2.imdecode`/`cv2.imencode` + Python 原生 `open()`。

### 6.5 中文文字渲染

**问题：** `cv2.putText` Hershey 字体只支持 ASCII。

**解决：** [utils/face_utils.py:48](utils/face_utils.py) 跨平台查找系统中文字体后用 Pillow 渲染。

## 七、模型转换（龙芯平台部署）

[scripts/convert_yolo.py](scripts/convert_yolo.py) 实现完整转换流程：

```
PyTorch .pt 模型
      ↓ torch.onnx.export()
  ONNX 模型 (.onnx)
      ↓ onnxruntime.quantization.quantize_dynamic()
  INT8 量化模型 (_int8.onnx)
      ↓ scp 部署到龙芯平台
  龙芯 2K1000 + onnxruntime CPU
```

**使用命令：**
```bash
# 推荐用于龙芯平台（轻量模型 + 量化）

# 开发环境测试
```

## 十、总结

本项目完成了一个功能完整的嵌入式人脸识别门禁系统：

- **3355 行 Python** + **1359 行 C 驱动** = 约 4700 行总代码
- **YOLOv5 四级回退**检测策略，兼容开发机(Windows+GPU)到部署机(龙芯 MIPS+ONNX)

- **PyQt5 多线程架构**，5个工作线程互斥运行，UI 流畅
- **SQLite 数据库**存储用户/特征/记录/配置
- **平台自适应语音**：Windows SAPI / Linux espeak
- **现代化 UI**：Catppuccin 暗色护眼主题，三标签页 + 统计卡片 + 详情面板
- **安全性设计**：陌生人报警 + 失败锁定 + 体温筛查
- **目标平台龙芯 2K1000**，国产化嵌入式部署，模型经 ONNX 量化适配
