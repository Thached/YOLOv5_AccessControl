## 概述

### 部署架构

```
┌──────────────────────┐         ┌──────────────────────────────┐
│    Ubuntu 24 开发机    │         │     龙芯 2K1000 (MIPS64el)    │
│                       │  scp    │                              │
│  ① 导出 ONNX 模型     │ ──────→ │  Python 3.x                  │
│  ② 交叉编译 ORT       │ ──────→ │  onnxruntime (预编译 .whl)    │
│  ③ 交叉编译 C 驱动    │ ──────→ │  SensorControl.so (自编译)    │
│  ④ 传输项目源码       │ ──────→ │  main.py / core / ui / utils │
└──────────────────────┘         └──────────────────────────────┘
```

### 开发机要求

- Ubuntu 24.04 LTS (amd64)
- Python 3.8+
- 已安装 ultralytics: `pip install ultralytics`
- 网络可访问 GitHub

### 龙芯平台要求

- 龙芯 2K1000 处理器（MIPS64 little-endian）
- Loongnix 或 Debian (mipsel) 操作系统
- Python 3.6+
- USB 摄像头
- 舵机、MLX90614、CS100A 传感器（可选，无硬件时自动模拟）


### 1.2 导出 ONNX 模型

```bash
# 标准导出（640×640 输入）
python scripts/convert_yolo.py --model yolov5su

# 低分辨率导出（320×320，龙芯推荐，推理快 4 倍）
python scripts/convert_yolo.py --model yolov5su --img-size 320

# INT8 量化导出（最小体积 + 最快速度，龙芯强烈推荐）
```

### 1.4 重要：INT8 模型部署说明

`face_detector.py` 默认加载 `model/yolov5su.onnx`，不会自动使用 INT8 版本。如果要在龙芯上使用 INT8 量化模型，有两种方式：

**方式一：重命名文件（推荐，无需改代码）**
```bash
# 备份原文件，然后用 INT8 版本覆盖
cp model/yolov5su.onnx model/yolov5su_fp32.onnx
cp model/yolov5su_int8.onnx model/yolov5su.onnx
```

**方式二：修改代码**
```python
# 编辑 core/face_detector.py 第 65 行，将:
onnx_path = f'model/{self.MODEL_NAME}.onnx'
# 改为:
onnx_path = f'model/{self.MODEL_NAME}_int8.onnx'
```

## 步骤二：PC 端交叉编译 onnxruntime

> **说明：** onnxruntime 官方不提供 MIPS64 预编译包，需要从源码交叉编译。
> 以下基于 onnxruntime v1.18 版本。在 Ubuntu 24 上可直接使用系统编译工具链。
> 如编译困难，可用备选方案（见[常见问题](#常见问题)）。

### 2.1 安装编译依赖

```bash
# 安装 CMake 和基础编译工具
sudo apt-get install -y cmake build-essential git python3-dev

# 确保交叉编译器已安装（见前置准备）
mips64el-linux-gnuabi64-gcc --version
```

### 2.2 配置交叉编译工具链

```bash
# 创建 CMake 工具链文件
cat > ~/mips64el_toolchain.cmake << 'EOF'
SET(CMAKE_SYSTEM_NAME Linux)
SET(CMAKE_SYSTEM_PROCESSOR mips64el)

SET(CMAKE_C_COMPILER   mips64el-linux-gnuabi64-gcc)
SET(CMAKE_CXX_COMPILER mips64el-linux-gnuabi64-g++)

SET(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
SET(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
SET(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)
SET(CMAKE_FIND_ROOT_PATH_MODE_PACKAGE ONLY)
EOF
```

> 如果使用的是龙芯官方工具链（前缀为 `mips64el-linux-gnu-`），请将上面文件中的 `mips64el-linux-gnuabi64` 替换为 `mips64el-linux-gnu`。

### 2.3 克隆并编译

```bash
# 克隆 onnxruntime
cd ~
git clone --depth 1 --branch v1.18.0 \
    https://github.com/microsoft/onnxruntime.git
cd onnxruntime

# 编译（MinSizeRel 减小体积，适合嵌入式）
./build.sh \
    --config MinSizeRel \
    --build_shared_lib \
    --parallel $(nproc) \
    --compile_no_warning_as_error \
    --skip_tests \
    --cmake_extra_defines CMAKE_TOOLCHAIN_FILE=~/mips64el_toolchain.cmake \
    --cmake_extra_defines onnxruntime_BUILD_SHARED_LIB=ON \
    --cmake_extra_defines onnxruntime_BUILD_UNIT_TESTS=OFF \
    --cmake_extra_defines onnxruntime_ENABLE_PYTHON=ON \
    --cmake_extra_defines onnxruntime_MINIMAL_BUILD=ON \
    --build_dir build_mips64

# 编译完成后生成 Python wheel
cd build_mips64/MinSizeRel
python setup.py bdist_wheel
```

编译产物位置：
```
build_mips64/MinSizeRel/dist/
└── onnxruntime-1.18.0-cp39-cp39-linux_mips64el.whl
```

### 2.4 备选方案

如果交叉编译 onnxruntime 过于复杂，可考虑以下替代方案：

**方案 A：在龙芯上原生编译（推荐，免去交叉编译复杂度）**
```bash
# 直接在龙芯平台上编译（耗时长但无需交叉工具链）
git clone https://github.com/microsoft/onnxruntime
cd onnxruntime
./build.sh --config Release --build_shared_lib --parallel 2 \
    --compile_no_warning_as_error --skip_tests
```

**方案 B：使用 OpenCV DNN 加载 ONNX**

修改 `face_detector.py`，在 `_try_load_onnx` 方法（第 61 行）之后添加 OpenCV DNN 后端：
```python
def _try_load_opencv_dnn(self):
    import cv2
    onnx_path = f'model/{self.MODEL_NAME}.onnx'
    if not os.path.exists(onnx_path):
        return False
    self.cv_net = cv2.dnn.readNetFromONNX(onnx_path)
    self.backend = 'opencv_dnn'
    return True
```

同时在 `_load_model` 方法中增加回退逻辑：
```python
def _load_model(self):
    if self._try_load_ultralytics():
        self.backend = 'ultralytics'
        return
    if self._try_load_onnx():
        self.backend = 'onnx'
        return
    # 新增 OpenCV DNN 回退
    if self._try_load_opencv_dnn():
        self.backend = 'opencv_dnn'
        return
    raise RuntimeError(...)
```

## 步骤三：PC 端交叉编译硬件驱动

### 3.1 确认 C 源码完整

C 驱动源码位于 `C_driver/` 目录下：

```
C_driver/
├── gpio.c                        # GPIO 控制（/dev/mem 内存映射）
├── gpio.h                        # GPIO 寄存器地址定义（基址 0x1fe10000）
├── main.c                        # 测试程序
├── i2c/
│   ├── smbus.c / smbus.h         # I2C SMBus 协议实现
│   └── i2cbusses.c / i2cbusses.h # I2C 总线管理
└── src/
    ├── SensorControl.c / .h      # 传感器统一控制接口
    ├── MLX90614.c / .h           # 红外测温（MLX90614, I2C 0x5A）
    ├── cs100a.c / .h             # 超声波测距（CS100A, GPIO）
    └── servo.c / .h              # 舵机控制（FS90, GP7101 DAC I2C 0x58）
```

```bash
# 验证源文件存在
ls -la C_driver/gpio.c C_driver/gpio.h
ls -la C_driver/src/SensorControl.c C_driver/src/MLX90614.c
ls -la C_driver/src/cs100a.c C_driver/src/servo.c
ls -la C_driver/i2c/smbus.c C_driver/i2c/i2cbusses.c
```

### 3.2 编译为 MIPS64 共享库

```bash
cd ~/YOLOv5_AccessControl/C_driver

# 编译所有 C 源文件为共享库
mips64el-linux-gnuabi64-gcc -shared -fPIC -O2 -o SensorControl.so \
    gpio.c \
    i2c/smbus.c i2c/i2cbusses.c \
    src/SensorControl.c src/MLX90614.c src/cs100a.c src/servo.c \
    -I. -Isrc -Ii2c

# 剥离调试符号以减小体积
mips64el-linux-gnuabi64-strip SensorControl.so

# 移动到 app 目录
mv SensorControl.so ../app/

# 验证产物
file ../app/SensorControl.so
# 应显示: ELF 64-bit LSB shared object, MIPS, MIPS64el ...
```

### 3.3 备选：在龙芯上直接编译

如果交叉编译出现问题，可以将 `C_driver/` 源码一同传输到龙芯，在龙芯上本地编译：

```bash
# 在龙芯平台上：
cd ~/YOLOv5_AccessControl/C_driver
gcc -shared -fPIC -O2 -o ../app/SensorControl.so \
    gpio.c i2c/smbus.c i2c/i2cbusses.c \
    src/SensorControl.c src/MLX90614.c src/cs100a.c src/servo.c \
    -I. -Isrc -Ii2c
```

```bash

    --exclude='C_driver' --exclude='scripts' --exclude='*.pt' \
    --exclude='data/*.db' \
    ~/YOLOv5_AccessControl/ root@${LOONGSON_IP}:~/YOLOv5_AccessControl/
```


### 6.3 首次运行

```bash
# 首次运行会自动创建 SQLite 数据库
python3 main.py
```

首次启动会：
1. 创建 `data/access_control.db` 数据库（含 users、face_features、access_records、settings 四张表）
2. 初始化 admin 账号（密码: admin123）
3. 自动检测并选择推理后端：
   - 有 PyTorch → ultralytics 后端
   - 有 onnxruntime → ONNX 后端
   - 都没有 → 报错退出
4. 状态栏显示: **"检测: YOLOv5su (ONNX)"**

### 6.4 启动自动运行

1. 点击菜单栏「管理」→「登录」
   - 账号: `admin`，密码: `admin123`
2. 点击「管理」→「自动运行」
3. 站在摄像头前（绿圈内），系统开始自动检测识别

自动运行流程：
1. 超声波测距 → 判断人是否在有效范围（默认 10-120cm）
2. 红外测温 → 判断体温是否正常（默认 < 37.2°C）
3. YOLOv5 检测人体 → 估算人脸区域
4. LBPH 特征比对 → 输出识别结果
5. 识别成功 → 舵机开门（默认延时 3 秒自动关闭）
6. 连续识别失败超过上限（默认 5 次）→ 系统锁定（默认 5 秒）

## 常见问题

### Q1: onnxruntime 交叉编译失败

**现象:** `./build.sh` 报错找不到交叉编译器

**解决:**
```bash
# 确认交叉编译器已安装
which mips64el-linux-gnuabi64-gcc

# 确认工具链文件中路径与实际一致
cat ~/mips64el_toolchain.cmake

# 或直接在龙芯平台上编译（更简单但耗时）
```

### Q2: 程序启动报 "无法加载 YOLOv5 模型"

**现象:** `RuntimeError: 无法加载 YOLOv5 模型`

**原因:** ultralytics 和 onnxruntime 都未安装，或 ONNX 文件不存在

**解决:**
```bash
# 确认 ONNX 文件存在
ls -la model/yolov5su.onnx

# 确认 onnxruntime 已安装
python3 -c "import onnxruntime; print('OK')"

# 如果缺少 onnxruntime，可在龙芯上直接安装 CPU 版本
pip3 install onnxruntime
```

### Q3: 摄像头无法打开

**现象:** 程序运行但视频画面黑屏

**解决:**
```bash
# 检查摄像头权限
sudo chmod 666 /dev/video0

# 或添加用户到 video 组（需重新登录生效）
sudo usermod -aG video $USER

# 测试摄像头
python3 -c "import cv2; cap=cv2.VideoCapture(0); print(cap.read()[0])"
```

### Q4: 硬件传感器不工作

**现象:** 硬件状态显示"模拟模式"

**说明:** 程序自动降级到模拟模式（`SimulatedHardware`），不影响识别功能，但道闸、测温、测距将使用随机模拟值。若需真实硬件：

```bash
# 1. 检查 I2C 设备
sudo i2cdetect -y 0

# 2. 检查 .so 文件是否为 MIPS64 格式
file app/SensorControl.so

# 3. 在龙芯上重新编译 C 驱动
cd ~/YOLOv5_AccessControl/C_driver
gcc -shared -fPIC -O2 -o ../app/SensorControl.so \
    gpio.c i2c/smbus.c i2c/i2cbusses.c \
    src/SensorControl.c src/MLX90614.c src/cs100a.c src/servo.c \
    -I. -Isrc -Ii2c
```

### Q5: PyQt5 无法安装

**现象:** `pip3 install PyQt5` 报错

**解决:**
```bash
# 优先使用系统包管理器
sudo apt-get install python3-pyqt5
```

### Q6: 推理速度太慢 (< 2 FPS)

**解决:**
1. 使用 320×320 的 ONNX 模型: `python scripts/convert_yolo.py --model yolov5su --img-size 320`
2. 使用 INT8 量化模型: `--quantize`（记得重命名为 `yolov5su.onnx` 或修改代码）
3. 增加检测间隔到 5 秒
4. 关闭不必要的系统服务释放 CPU

### Q7: 语音播报不工作

**现象:** 终端显示 `[语音] pyttsx3 不可用`

**解决:**
```bash
# 安装 espeak 和 pyttsx3
sudo apt-get install -y espeak python3-espeak
pip3 install pyttsx3
```

### Q8: 中文字体显示为方块

**现象:** 识别结果中文显示异常

**解决:**
```bash
# 安装中文字体
sudo apt-get install -y fonts-wqy-zenhei fonts-wqy-microhei
# 程序会自动扫描 /usr/share/fonts/ 查找支持中文的字体
```

## 快速命令汇总

```bash
# ===== Ubuntu 24 开发机 =====

# 0. 安装交叉编译工具链（一次性）
```
